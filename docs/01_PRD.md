AgentLoop — PRD (Product Requirements Document)

정부지원사업 계획서 작성을 AI Agent와 협업하기 위한 로컬 워크스페이스 도구


1. 배경 및 문제
현재 워크플로우

정부지원사업 계획서를 Claude 등 AI Agent와 협업하여 작성
프로젝트별 폴더(010_AI바우처소방적산 등)에 마크다운 문서를 쌓아가며 진행
3자리 코드 체계(XYZ_파일명.md)로 문서를 분류 관리
CLAUDE.md로 Agent에게 규칙 전달, 000_index.md로 현황 추적

해결하려는 문제

프로젝트 초기화가 쉘 스크립트 수동 실행 → UI로 간편화
문서 분류를 Agent가 매번 수동으로 → 자동 감지/분류
마크다운 파일이 에디터에서만 보임 → 깔끔한 뷰어 필요
프로젝트마다 반복되는 프롬프트 패턴 → 재사용 가능한 스킬로 관리


2. 제품 개요
AgentLoop는 로컬에서 실행되는 웹 애플리케이션으로, 정부지원사업 문서 작성 프로세스를 체계화하고 AI Agent 협업을 효율화하는 도구.

로컬 실행: localhost에서 동작, 문서는 파일시스템에 그대로 유지
기존 체계 위에 얹힘: 3자리 코드 체계, CLAUDE.md, index.md를 그대로 활용
AI Agent 연동: Agent가 만든 문서를 자동 인식, 스킬 기반 작업 지원


3. 사용자

정부지원사업 계획서를 작성하는 실무자 (gray 본인)
Claude 등 AI Agent와 함께 문서 작업을 수행하는 사람


4. 기능 정의
Phase 1 — 핵심 뼈대 (MVP)
F1. 프로젝트 대시보드

docs 폴더 내 프로젝트 폴더 목록 표시
각 프로젝트의 문서 수, 최근 수정일 요약
프로젝트 클릭 → 해당 프로젝트 상세 진입

F2. 프로젝트 초기화

UI 폼: 프로젝트 번호 + 이름 입력
실행 시 폴더 생성 + CLAUDE.md + 000_index.md 자동 생성
기존 init_project.sh 로직을 API로 구현

F3. 프로젝트 상세 뷰

index.md 기반 문서 현황 시각화 (대분류별 테이블)
작업 로그 타임라인
문서 클릭 → 마크다운 뷰어로 열기

F4. 마크다운 뷰어

마크다운 파일을 렌더링해서 깔끔하게 표시
테이블, 코드블록, 링크 등 지원
문서 간 이동 네비게이션

Phase 2 — 자동 분류
F5. 미분류 문서 감지

프로젝트 폴더 내 코드 체계를 따르지 않는 파일 감지
"미분류" 섹션에 표시
사용자가 직접 분류하거나, AI 분류 제안 수락

F6. 파일 워치 + 자동 index 업데이트

폴더 변경 감지 (watchdog)
새 파일 추가 시 → 대분류 추천 + index.md 자동 업데이트 제안

Phase 3 — 스킬 시스템
F7. 스킬 등록

프롬프트 + 메타데이터(대상 대분류, 입력/출력 형식) 묶어서 저장
예: "RFP 분석" 스킬 = 프롬프트 + 출력은 1xx 문서

F8. 스킬 실행

스킬 선택 → 입력 문서 지정 → 프롬프트 조합 → Agent에게 전달
결과물 자동 코드 부여 + index 등록


5. 비기능 요구사항

문서는 항상 파일시스템에 원본 유지 (DB 의존 없음)
로컬 전용, 외부 네트워크 불필요 (AI API 호출 제외)
docs 폴더 경로를 설정으로 지정 가능
