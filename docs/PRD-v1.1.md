요청하신 의견들을 모두 통합하여, 단순한 문서 뷰어를 넘어 **'AI 에이전트와의 매끄러운 협업(Copilot) 에디터'**로 진화한 **AgentLoop PRD v1.1**을 작성했습니다.

이전의 1-Step Navigation(Master-Detail) 구조와 Context Builder 등에 더해, 새롭게 논의된 **'인라인 피드백(마크다운 자동 주입)'** 기능을 핵심 차별화 요소로 배치했습니다.

# AgentLoop — PRD (Product Requirements Document) v1.1

## 1. 배경 및 문제

**현재 워크플로우의 한계**

* **잦은 화면 전환 (Context Switching):** 작성된 마크다운 문서를 확인하고, 에디터로 돌아가 줄 번호를 찾아서 코멘트(`!!`)를 남긴 뒤, 다시 Agent에게 지시하는 번거로운 과정 반복.
* **비효율적인 문서 탐색:** 목록 페이지와 상세 페이지를 오가는(2-Step) 구조로 인해 여러 문서를 빠르게 대조하기 어려움.
* **Agent 컨텍스트 구성의 번거로움:** "A문서와 B문서를 참고해서 C를 작성해"라는 프롬프트를 작성하기 위해 매번 문서 내용을 복사/붙여넣기 해야 함.
* **수정 이력 추적 불가:** Agent가 문서를 업데이트했을 때, 어느 부분이 수정되었는지 즉각적으로 파악하기 어려움.

**해결 방향**
로컬 파일시스템(마크다운) 기반을 유지하면서, 브라우저 위에서 즉각적인 **[문서 열람 ↔ 피드백 ↔ Agent 프롬프트 생성]**이 원스톱으로 이루어지는 1-Step UI/UX 통합 워크스페이스 구축.

---

## 2. 제품 개요

AgentLoop는 로컬 환경에서 실행되며, 정부지원사업 문서 작성 프로세스를 체계화하고 Claude 등 AI Agent와의 피드백 루프를 극도로 효율화하는 **AI 협업 전용 마크다운 관리 도구**입니다.

* **1-Step Navigation:** 좌측 사이드바(목록) + 우측 메인(뷰어/에디터) 구조로 탐색 비용 최소화.
* **Visual to Markdown Feedback:** 뷰어에서 텍스트를 드래그해 코멘트를 남기면, Agent가 인식할 수 있는 마크다운 문법으로 자동 치환되어 원본 파일에 주입됨.
* **Local & File-System First:** 별도 DB 없이 로컬 `docs/` 폴더 내의 `.md` 파일과 `000_index.md`를 단일 진실 공급원(SSOT)으로 활용.

## 5. 핵심 기능 정의

### Phase 1 — 코어 워크스페이스 (Master-Detail)

* **F1. 1-Step 대시보드 및 탐색:** 좌측에 프로젝트 목록 및 폴더 트리, 우측에 뷰어를 배치하여 화면 전환 없이 문서를 탐색.
* **F2. 프로젝트 관리 (초기화 및 정렬):** `000_index.md` 및 `CLAUDE.md` 자동 생성. 프로젝트 목록은 드래그 앤 드롭(DnD)으로 순서 변경 가능.
* **F3. 미분류 문서 자동 감지 (Orphan Detection):** `docs/` 폴더 내에 존재하지만 `index.md`에 등록되지 않은 파일을 스캔하여 사이드바 최상단 [미분류 문서]에 노출. 클릭 시 대분류 지정 및 index 자동 업데이트 지원.

### Phase 2 — AI 협업 피드백 루프 (핵심 차별화)

* **F4. 인라인 피드백 & 마크다운 자동 주입 (Visual to Markdown):**
* **동작:** 사용자가 우측 뷰어에서 수정이 필요한 텍스트를 마우스로 드래그 → 플로팅 버튼 `[💬 AI에게 수정 요청]` 클릭 → 자연어로 지시사항 입력.
* **자동 주입 로직:** Frontend가 선택된 텍스트의 AST(Abstract Syntax Tree) 라인 넘버를 추적하여, Backend를 통해 원본 `.md` 파일의 해당 줄 바로 아래에 **Agent 전용 코멘트 블록**을 삽입.
* **삽입 포맷 예시:**
```markdown
> 💡 **[사용자 피드백]**
> !! 타겟 텍스트: "국내 소방 안전 시장은 2021년 기준..."
> !! 지시사항: 이 부분 21년 통계 말고 26년 최신 통계로 바꿔서 다시 써줘.

```


* **시각화:** 뷰어에서는 삽입된 코멘트 블록을 눈에 띄는 색상(예: 노란색 배경)의 포스트잇 형태로 렌더링.


* **F5. 문서 비교 뷰어 (Diff Viewer):**
* 문서 간 변경 사항 추적 (예: `402_사업화_초안.md` vs `403_사업화_수정본.md`).
* Github Split View 형태로 좌/우 또는 인라인으로 추가/삭제된 텍스트를 하이라이팅하여 Agent의 작업 결과를 빠르게 검수.



### Phase 3 — 프롬프트 엔지니어링 자동화

* **F6. 문서 장바구니 (Context Builder):**
* 사이드바 문서 목록에 체크박스 제공.
* 여러 문서를 체크한 뒤 `[프롬프트 생성]` 버튼을 누르면, 체크된 문서들의 원본 마크다운 텍스트를 취합하여 Agent에게 전달하기 좋은 `<document>` XML 태그 포맷으로 자동 감싸서 클립보드에 복사.


* **F7. 스킬 템플릿 시스템:**
* 자주 쓰는 지시사항(예: "선택된 문서들을 바탕으로 평가항목에 맞춰 400번대 목차를 구성해")을 스킬로 저장하고 재사용.

## 6. 기술 설계 및 고려사항

* **Frontend:** React + TypeScript + Vite, Tailwind CSS
* 드래그 앤 드롭: `@hello-pangea/dnd`
* 마크다운 렌더링 및 AST 추적: `react-markdown`, `remark`, `rehype` 생태계 활용 (HTML 렌더링 시 `data-source-line` 속성 삽입 필수).
* Diff 뷰어: `react-diff-viewer`


* **Backend:** FastAPI (Python 3.10+)
* 저장소: 로컬 파일시스템 직접 I/O.
* API: 특정 라인 넘버에 텍스트를 `insert` 하는 API 추가 필요.



---

이 기획안은 단순히 '잘 보여주는 것'을 넘어, **사용자의 브라우저 상 행동(드래그, 코멘트)이 Agent를 제어하기 위한 데이터(마크다운 프롬프트)로 자동 변환**된다는 점에서 강력한 도구가 될 것입니다.
