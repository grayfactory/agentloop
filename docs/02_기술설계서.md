3. Backend API 설계
3.1 프로젝트 API
MethodEndpoint설명GET/api/projects프로젝트 목록 (폴더명, 문서수, 최근수정일)GET/api/projects/{name}프로젝트 상세 (index.md 파싱 결과)POST/api/projects프로젝트 초기화 (폴더+CLAUDE.md+index 생성)
3.2 문서 API
MethodEndpoint설명GET/api/projects/{name}/documents문서 목록 (코드별 그룹핑)GET/api/projects/{name}/documents/{filename}문서 내용 (raw markdown)GET/api/projects/{name}/worklog작업 로그 목록
3.3 설정 API
MethodEndpoint설명GET/api/config현재 설정 조회PUT/api/configdocs 경로 등 설정 변경

4. 핵심 데이터 모델
4.1 프로젝트 (from 폴더 스캔)
pythonclass Project(BaseModel):
    folder_name: str        # "010_AI바우처소방적산"
    project_num: str        # "010"
    project_title: str      # "AI바우처소방적산"
    doc_count: int          # 문서 수
    last_modified: datetime # 최근 수정일
4.2 문서 (from index.md 파싱)
pythonclass Document(BaseModel):
    code: str               # "000", "100", "201"
    filename: str           # "100_RFP원문정리.md"
    summary: str            # 30자 이내 요약
    status: str             # "운영중", "완료", "진행중"
    category: int           # 대분류 0~9
4.3 작업 로그 (from index.md 하단)
pythonclass WorkLog(BaseModel):
    date: str               # "2026-03-11"
    content: str            # 50자 이내 작업 내용
    related_docs: str       # 관련 문서

5. index.md 파싱 전략
index.md는 마크다운 테이블로 구성되어 있으므로, 정규식 기반으로 파싱:
### {카테고리 헤더}
| 코드 | 파일명 | 요약 | 상태 |
|------|--------|------|------|
| {code} | {filename} | {summary} | {status} |

카테고리 헤더에서 대분류 번호 추출: ### 1xx RFP/공고 분석 → category=1
테이블 행에서 문서 정보 추출
작업 로그 섹션은 ## 작업 로그 이후 테이블에서 추출


6. Frontend 핵심 컴포넌트
6.1 Dashboard (프로젝트 목록)

GET /api/projects 호출 → ProjectCard 리스트 렌더링
우측 상단 [+ 새 프로젝트] → InitProjectModal 열기

6.2 Project (프로젝트 상세)

좌측 사이드바: 대분류별 문서 목록 (DocumentList)
우측: 선택된 문서의 마크다운 렌더링 (MarkdownViewer)
하단 또는 사이드바 하단: 작업 로그 (WorkLog)

6.3 MarkdownViewer

react-markdown + remark-gfm 으로 GFM 지원 (테이블, 체크박스 등)
rehype-highlight로 코드 하이라이팅
커스텀 스타일링 (깔끔한 문서 뷰)
